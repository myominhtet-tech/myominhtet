  • Introduction Responsive Design

    ယေဘူရအားဖြင့် responsive design တွေမှာ fluid grid တွေ fluid images တွေနဲ့ media queries တွေပါဝင်ပါတယ်။ကျွန်တော်တို့ဟာ responsive design အချို့ကိုလေ့လာပြီးဖြစ်ပါတယ်။ဒါ့ကြောင့်ကျွန်တော်တို့ဟာအခြေခံ concept တွေနဲ့နည်းလမ်းတွေကိုနားလည်ခဲ့ပြီးသားဖြစ်ပြီးတော့အခု advanced concept တွေပိုပြီးလေ့လာရမှဖြစ်ပါတယ်။ဒီသင်ခန်းစာမှာတော့ကျွန်တော်တို့ responsive design တွေမှာရှိတဲ့ Layout တွေကိုလေ့လာရမှာပဲဖြစ်ပါတယ်။responsive design တွေရဲ့နောက်မှာရှိတဲ့အခြေခံ theory တွေကိုလေ့လာရမှာဖြစ်ပါတယ်။breakpoints တွေ design patterns တွေလိုမျိုး specific concepts တွေကိုတစ်ဆင့်ချင်းလေ့လာရမှာဖြစ်ပါတယ်။Rails app တွေ php site တွေ wordPress theme တွေ static website တွေနဲ့ အခြား type တွေနဲ့တည်ဆောက်ထားတဲ့weppage တွေနဲ့နည်းလမ်းတွေကတူညီပါတယ်။

    responsive web design တွေက screen sizes တွေအမျိုးမျိုးပေါ်မှာအလုပ်လုပ်မဲ့ websites တွေတည်ဆောက်တဲ့အခါမှာအသုံးပြုဖို့အတွက် techniques တွေကိုစုစည်းထားတာပဲဖြစ်ပါတယ်။responsive design တွေရဲ့scope တွေကတစ်ခါတစ်ရံမှာ traditional screens သို့မဟုတ့်high DPI screens တွေလိုမျိုး မတူညီတဲ့ pixel densities တွေမှာအလုပ်လုပ်ဖို့ပါဝင်ပါတယ်။အဲ့တာက 2010 May မှာစခဲပြီး A List Apart ဆိုတဲ့ blog ပေါ်မှာ Responsive Web Design ရဲ့seminal article တွေကိုရေးခဲ့ပါတယ်။Web professional အများစုက sites တွေကိုဖန်တီးတဲ့အခါမှာ desktop version အတွက်နဲ့ mobile version အတွက်ဆိုပြီးနှစ်မျိုးဖန်တီးကြပါတယ်။စိချရမှုမရှိတဲ့ browser တွေကိုသုံးတဲ့အခါကာကွယ်ဖို့အတွက်လူတွေက javascript ကိုလည်းအသုံးပြုကြပါတယ်။phone sizes တွေပိုများလာပြီး tablets တွေပါအများကြီးပေါ်ထွက်လာတဲ့အခါ sites တွေက layout တွေမတည်ညိမ်တော့ပါဘူး။အဲ့တာတွေကိုဖြေရှင်းနိုင်မဲနည်းလမ်းကို Ethan ကရှာဖွေပြီးတော့ မတူညီတဲ့ device တွေ screen size တွေရဲ့ Layout တွေအတွက် layout တွေတည်ဆောက်ဖို့ responsive web design ဆိုတာကိုဖော်ထုပ်ခဲ့ကြပါတယ်။

    web designers အများစုကကြိုတင်ပြီးရေးဆွဲထားတဲ့ပုံစံတွေအတိုင်း responsive web design ကိုပြုလုပ်ကြပါတယ်။layout တွေကို fixed pixel-based units တွေနဲ့အတိအကျထိမ်းချုပ်ထားပါတယ်။ရည်ရွယ်ချက်ကတော့ browser တိုင်းမှာကျွန်တော်တို့ရဲ့ websites ကတိတိကျကျတူညီနေအောင်ပြုလုပ်ပေးရမှာပဲဖြစ်ပါတယ်။digital screen တွေကတော့ Print pages တွေနဲ့မတူညီပါဘူး။သူတို့က မတူတဲ့ sizes အမျိုးမျိုးနဲ့လာပြီးတော့ကွဲပြားစွာ oriented လုပ်နိုင်ပါတယ်။ပြီးတော့သူတို့မှာ ကွဲပြားတဲ့ pixel densities တွေရှိပါတယ်။ကျွန်တော်တို့တွေဟာ layout တွေအတွက်ကိုက်ညီတဲ့ values တွေကိုထည့်ပေးမဲ့အစား ratios သို့မဟုတ် percentage တွေကိုအသုံးပြုနိုင်ပါတယ်။ကျွန်တော်တို့ရဲ့ဥပမာထဲမှာတော့ ရိုးရှင်းတဲ့ two-column layout ကိုဖော်ပြထားပြီးဘယ်လိုပြောင်းလဲတယ်ဆိုတာတွေကိုလေ့လာရမှာပဲဖြစ်ပါတယ်။responsive design မဖြစ်ခင်မှာ layout က 960pixelsရှိပြီးတော့ screen ရဲ့အလယ်မှာတည်ရှိပါတယ်။ပထမ column မှာတော့ 672 pixels ရှိပြီးတော့ဒုတိယ column မှာတော့ 288 pixels ရှိပါတယ်။အဲ့တာကို photoshop ထဲမှာဆွဲပြီးတော့ CSS ထဲမှာပြန်ထည့်နိုင်ပါတယ်။ဒီနေ့အချိန်တွေမှာတော့ကျွန်တော်တို့က အခြေခံ wire frames တွေကိုအသုံးပြုပါတယ်။အဲ့တာတွေက element တစ်ခုနဲ့တစ်ခုကိုအချိုးကျအောင်ဆက်နွယ်ပေးပါတယ်။ကျွန်တော်တို့ရဲ့ CSS ထဲမှာတော့ percentages or ems ဆိုတဲ့ relative units တွေကိုအသုံးပြုနိုင်ပါတယ်။ဥပမာထဲမှာရှိတဲ့ကျွန်တော်တို့ရဲ့ left column မှာ 70% area ထားပြီးတော့ right column မှာတော့ 30% ပေးထားခဲ့ပါမယ်။elements တွေမှာ percentage တွေအသုံးပြုတာကသူတို့ရဲ့ parent container တွေနဲ့ဆက်နွယ်နေပါတယ်။ဆိုလိုတာကတော့ ကျွန်တော်တို့ဟာ parent ပေါ်မှာအခြား percentage တွေသတ်ရောက်ဖို့အတွက်နဲ့ ကျွန်တော်တို့က column နှစ်ခုကြားမှာရှိတဲ့အချိုးကိုထိန်းညှိပေးရမှာပဲဖြစ်ပါတယ်။ကျွန်တော်တို့က parent ကို 80% ပေးခဲ့မယ်ဆိုရင်အဲ့တာက document window ဖြစ်ပြီးတော့ browser width ရဲ့ 80% ကိုယူထားတာပဲဖြစ်ပါတယ်။တစ်ချို့အခါတွေမှာ ဆိုရင်percentages တွေကအကြာကြီးထိန်းထားနိုင်မှာမဟုတ်ပါဘူး။ကျွန်တော်တို့တွေ layout တွေကိုက်ညီအောက်ပြန်ပြီးချိန်ညှိပေးဖို့လိုအပ်ပါတယ်။multi-column layout တွေက screen အကြီးတွေပေါ်မှာကြည့်လို့အဆင်ပြေပြီးတော့ phone ပေါ်မှာဆိုရင်တော့ single-column layout ကိုပဲအသုံးပြုသင့်ပါတယ်။အဲ့တာတွေကြောင့်ကျွန်တော်တို့က Media queries တွေကိုလိုအပ်ပါတယ်။

  • Using Media Queries In Responsive Web Design

    ကျွန်တော်တို့ဟာ media queries တွေကိုအချို့project တွေမှာအသုံးပြုခဲ့ဘူးပါတယ်။media queries တွေကိုနားလည်ခြင်းမရှိပဲနဲ့ responsive web design တွေအကြောင်းကိုပြောဖို့ရန်အတွက်ခက်ခဲ့ပါလိမ့်မယ်။အမှန်တကယ်ကတော့ media queries တွေဟာ css rules တွေဖြစ်ပြီး သေချာတဲ့အခြေအနေကိုတွေ့တဲ့အခါ CSS code တွေကိုအသုံးပြုဖို့ကူညီပေးပါတယ်။အဲ့တာတွေကို media features လို့ခေါ်ပါတယ်။responsive web design တွေမှာရှိတဲ့ media features တွေမှာဆိုရင် min width တွေကိုအများဆုံးအသုံးပြုပါတယ်။ကျွန်တော်တို့ဟာ media features တွေအများကြီးကို MDN documentation ထဲမှာလေ့လာနိုင်ပါတယ်။ကျွန်တော်တို့ example ထဲမှာတော့ ရိုးရှင်းတဲ့ html တွေနဲ့ media features တွေကိုရေးပြထားပါတယ်။example ကို browser ပေါ်မှာ run ပြီးအချုံ့အချဲ့လုပ်ကြည့်တဲ့အခါမှာ media features တွေအလုပ်လုပ်တာကိုတွေ့ရမှာပဲဖြစ်ပါတယ်။

    ကျွန်တော်တို့ဒီတစ်ခါမှာတော့ responsive navigation အကြောင်းပြောပြသွားမှာဖြစ်ပါတယ်။CSS media queries နဲ့ Flexbox တွေကိုအသုံးပြုတာပဲဖြစ်ပါတယ်။အဲ့တာကြောင့် navigation က website တွေရဲ့အရေးကြီးတဲ့အပိုင်းတစ်ခုပဲဖြစ်ပါတယ်။ဘာ့ကြောင့်လဲဆိုတော့အဲ့တာကကျွန်တော်တို့ site မှာရှိတဲ့ areas တွေကို direct သွားနိုင်ပြီးကျွန်တော်တို့လိုချင်တဲ့နေရာကိုတန်းပြီးရောက်နိုင်ပါတယ်။အခုအချိန်မှာတော့ websites တွေအများစုမှာ navigtations တွေက viewport sizes နဲ့ devices တွေအမျိုးမျိုးမှာတူညီတဲ့ပုံစံတွေနဲ့အလုပ်မလုပ်တော့ပါဘူး။အသုံးပြုတဲ့သူတွေကလည်းသူတို့အသုံးပြုတဲ့ screen size အရ navigation တွေအအလုပ်လုပ်တာမျိုးတွေကိုလိုအပ်ပါတယ်။တည်ဆောက်တဲ့အခါမှာကျွန်တော်တို့ mobile-first layout အတွက်စပြီးတည်ဆောက်ရမှာဖြစ်ပါတယ်။
