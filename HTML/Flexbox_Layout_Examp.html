  • Introduction

    CSS Flexbox layout တွေထဲမှာကျွန်တော်တို့ webpages တွေအတွက်ပိုပြီးထူးခြားတဲ့ method တစ်ခုကိုလေ့လာရမှာပဲဖြစ်ပါတယ်။CSS lab methods တွေက floats တွေလိုဖြစ်ပြီးတော့ inline block နဲ့ absolute positioning တွေမှာတိကျတဲ့ကန့်သတ်ချက်တွေရှိပါတယ်။ယနေ့ခေတ်ရှုပ်ထွေးတဲ့ responsive websites တွေရဲ့ layout တွေကိုကိုယ်တိုင်ပြုပြင်ဖို့အတွက်ကျွန်တော်တို့က design တွေရှိဖို့မလိုအပ်ပါဘူး။ကျွန်တော်တို့ CSS Layout Basics တွေကိုလေ့လာတဲ့အခါမှာ floats ကပြသနာဖြစ်နိုင်ပါတယ်။ဘာ့ကြောင့်လဲဆိုတော့သူတို့ရဲ့ထူးခြားတဲ့အပြုအမူကြောင့်ဖြစ်ပါတယ်။collapsing ကနေ elements တွေပါဝင်နေတာကိုကာကွယ်ဖို့ရန်အတွက်ပြုပြင်ချင်းတွေကိုရှင်းလင်းထားဖို့လိုအပ်ပါလိမ့်မယ်။ကျွန်တော်တို့ဟာ display မှာ inline block ကိုအသုံးပြုတဲ့အခါ browser က default သတ်မှတ်ထားတဲ့အဖြူရောင်နေရာလွတ်နဲ့အမြဲတမ်းအဆင်မပြေဖြစ်နေတာကိုတွေ့ရမှာပဲဖြစ်ပါတယ်။ page layout မှာဘယ်တည်နေရာကိုမဆို tricky လုပ်နိုင်ပါတယ်။ဘာ့ကြောင့်လဲဆိုတော့ အဲ့တာက context တွေရဲ့တည်နေရာတွေ ၊positioned elements တွေတောင်မရှိပဲ page ပေါ်မှာလုပ်ဆောက်နေတဲ့အခြား content တွေကိုလိုအပ်ပါတယ်။ အဲ့တာကြောင့် Flexbox (သို့) Flexible Box Layout က css properties ရဲ့အစုတွေပဲဖြစ်ပါတယ်။ အဲ့တာတွေက page content ရဲ့ layout ကို flexible way တွေထောက်ပံ့ပေးပါတယ်။ အဲ့တာက ထူးခြားတဲ့ layout feature ဖြစ်ပြီး web designers တွေက layout တွေကိုာဘယ်လိုတည်ဆောက်တယ်ဆိုတာကိုပြန်ပြီးသတ်မှတ်နိုင်ပါတယ်။stacking elements တွေကဲ့သို့သော floats, block, inline block တွေရဲ့ကောင်းကျိုးကိုကျွန်တော်တို့ရရှိနိုင်ပါတယ်။အဲ့တာတွေကိုအသုံးပြုတဲ့အခါမှာဖြစ်လာတဲ့ browser ရဲ့တစ်မူထူးခြားတာတွေမရှိပဲနဲ့ တစ်ဖက်နဲ့တစ်ဖတ်မှီခိုပြီးတော့နေနိုင်ပါတယ်။မူလ size တွေ HTML ထဲမှာရှိတဲ့ order တွေကိုဂရုမစိုက်ပဲ navigation links ,content columns တွေနဲ့ images တွေကဲ့သို့သောအရာတွေကို flexbox ကိုသုံးပြီးတော့ direction, size ,order သို့မဟုတ် elements တွေကိုကျွန်တော်တို့ပြောင်းလဲနိုင်ပါတယ်။ကျွန်တော်တို့ content တွေကို container ထဲမှာဘယ်နေရာမှာပဲဖြစ်ဖြစ်ထားနိုင်ရန်အတွက်အဲ့တာကပေးပြီးရိုးရှင်းတဲ့ intelligent tools ပဲဖြစ်ပါတယ်။source code ထဲမှာသူတို့ပေါ်လာတဲ့နေရာမှာတော့ပြသနာမရှိပါဘူး။flexbox တွေပြုလုပ်ပြီး responsive layouts တွေတည်ဆောက်ခြင်းက သေးငယ်တဲ့ mobile devices တွေကနေ ကြီးမားတဲ့ desktop screens တွေအထိအမြဲတမ်းလွယ်ကူနေပါလိမ့်မယ်။

  • Concept of Flexbox

    ကျွန်တော်တို့ရဲ့ layout ထဲမှာ flexbox တွေကိုအသုံးမပြုခင်မှာflexbox နောက်မှာရှိတဲ့ terminology တွေ concept တွေကိုနားလည်ထားဖို့လိုအပ်ပါလိမ့်မယ်။flexbox layout ထဲမှာအရေးအကြီးဆုံး element နှစ်ခုကတော့ flex container နဲ့ flex items တွေပဲဖြစ်ပါတယ်။ပထမ အဆင့်အနေနဲ့ကျွန်တော်တို့ရဲ့ layout တွေထဲမှာ flexbox တွေကိုအသုံးပြုမယ်ဆိုရင် flex container ကိုတည်ဆောက်ပေးဖို့လိုအပ်ပါတယ်။flex container က flexbox layout ရဲ့အပိုင်းတစ်ခုဖြစ်ပြီးတော့သူ့ထဲမှာတော့ flex items တွေပါဝင်ပါတယ်။flex container က div,ul,header တွေလိုမျိူး block-level တစ်ခုခု (သို့မဟုတ်) inline element တစ်ခုဖြစ်လာနိုင်ပါတယ်။ flex container ထဲမှာရှိတဲ့ direct child တွေကို flex item လို့ခေါ်ပါတယ်။ဥပမာကျွန်တော်တို့မှာ unorder list တစ်ခုရှိတယ်ဆိုရင် ul ဆိုတဲ့ tag က flex container ဖြစ်ပြီးတော့ li tag တွေကတော့ flex items တွေပဲဖြစ်ပါတယ်။ကျွန်တော်တို့က flexbox ရဲ့ powerful aligement properties တွေကိုအသုံးပြုပြီး flex items တွေရဲ့တည်နေရာတွေကိုပြောင်းလဲနိုင်ပါတယ်။နောက်ထပ် flexbox တွေဘယ်လိုအလုပ်လုပ်သလဲဆိုတာကိုလည်းကျွန်တော်တို့သိထားဖို့လိုအပ်ပါတယ်။browser တွေမှာ axis တစ်လျှောက်ဘယ်လိုနေရာခြထားနိုင်တယ်ဆိုတာကိုလည်းသိဖို့လိုအပ်ပါတယ်။flexbox တွေမှာ axis နှစ်ခုရှိပြီးတော့ flex items တွေရဲ့ Layout direction တွေကိုသတ်မှတ်ပေးတာပဲဖြစ်ပါတယ်။container ထဲမှာ items တွေဟာ main axis (or) cross axis အတိုင်းတည်ရှိပါတယ်။main axis ကတော့မူလနေရာဖြစ်ပြီး item တွေကပုံမှန်အတိုင်းဆိုရင်main axis အတိုင်းတည်ရှိပါတယ်။cross axis ကတော့ Main axis ပေါ်မှာဒောင့်မတ်ကြစွာတည်ရှိပါတယ်။ axis တစ်ခုစီတိုင်းမှာ start side နဲ့ end side ဆိုပြီးရှိပါတယ်။main axis မှာရှိတဲ့ default main start နဲ့ main end တွေရဲ့ဦးတည်ချက်ကတော့ဘယ်ဘက်ကနေညာဘက်ကိုသွားတာဖြစ်ပြီး cross axis ရဲ့ default direction ကတော့ အပေါ်ကနေအောက်ကိုသွားတာပဲဖြစ်ပါတယ်။ဒါပေမယ့်ကျွန်တော်တို့က direction တွေကိုလွယ်လွယ်ကူကူပဲပြောင်းလဲနိုင်ပါတယ်။

  • Creating Flexbox

    ကျွန်တော်တို့အခြေခံ concepts တွေ terminology တွေအကြောင်းကိုလေ့လာခဲ့ပြီးဖြစ်ပါတယ်။အဲ့တာကြောင့်ပထမအဆင့်အနေနဲ့flexbox layout တွေတည်ဆောက်ဖို့ flex container တွေသတ်မှတ်ဖို့အတွက်ကျွန်တော်တို့ လေ့လာရမှာပဲဖြစ်ပါတယ်။ဥပမာထဲမှာကျွန်တော်တို့ရေးပြထားသလိုပဲ container ဆိုတဲ့ div element တစ်ခုတည်ဆောက်ပြီး items တွေထည့်ရေးထားပါတယ်။Flexbox layout အတွက်ကျွန်တော်တို့ css style file သီးခြားတည်ဆောက်ပြီးရေးပြထားပါတယ်။ကျွန်တော်တို့က flexbox property တွေမရေးခင်မှာ layout ထဲမှာ flex container ကိုသတ်မှတ်ခဲ့ဖို့လိုအပ်ပါတယ်။အဲ့တာကြောင့်ကျွန်တော်တို့က container selector ထဲမှာ display property ထဲမှာ value ကို flex လို့ပေးခဲပြီး flex container ကိုသတ်မှတ်ခဲ့ပါတယ်။အဲ့လိုသတ်မှတ်လိုက်တဲ့အခါမှာ container ထဲမှာရှိတဲ့ items တွေက flex items တွေဖြစ်သွားပါတယ်။

  • Flexbox Direction Properties

    ပြီးခဲ့တဲ့သင်ခန်းစာမှာကျွန်တော်တို့ flex (သို့) inline flex ဆိုတဲ့တန်ဖိုးတွေကို display မှာပေးပြီးလေ့လာခဲ့ကြပါတယ်။container က flex box layout အတွက် context တွေပါဝင်တဲ့အစုတစ်ခုပဲဖြစ်ပြီး flex items တွေလည်းပါဝင်ပါတယ်။flex container တွေကိုသတ်မှတ်တဲ့အခါမှာ ကျွန်တော်တို့စပြီး flex boxရဲ့ထူးခြားချက်တွေကိုတွေ့ရမှာဖြစ်ပါတယ်။အဲ့တာတွေကတော့ powerful direction order, alignment နဲ့ sizing ဆိုတဲ့ properties တွေပဲဖြစ်ပါတယ်။အချို့ flex box properties တွေက flex container ကိုပဲသတ်ရောက်တာဖြစ်ပြီးအချို့တွေကတော့ flex items တွေပေါ်မှာပဲသတ်ရောက်တာဖြစ်ပါတယ်။အဲ့တာကြောင့်ကျွန်တော်တို့အရေးတကြီးသိရမှာကတော့ ဘယ်properties က flex containers ပေါ်မှာသတ်ရောက်တယ် flex items တွေပေါ်မှာသတ်ရောက်တယ်ဆိုတာတွေပဲဖြစ်ပါတယ်။တစ်နည်းအားဖြင့် flex items တွေကကျွန်တော်တို့ရဲ့ Layout တွေပေါ်မှာသတ်ရောက်မှုမရှိပဲ Properties တွေကိုအသုံးပြုပြီးအဆုံးသတ်နိုင်ပါတယ်။ပထမကျွန်တော်တို့ flex direction property ကိုအသုံးပြုပြီးတော့ flex box layout တွေရဲ့ direction တွေကိုထိမ်းချုပ်နိုင်ပါတယ်။အောက်မှာပြထားတဲ့ဥပမာထဲမှာတော့ flex direction ကို vertical column layout ကိုပြောင်းပြပေးထားတာဖြစ်ပါတယ်။flex direction property က flex container ကိုပဲသတ်ရောက်တာဖြစ်ပါတယ်။flex items တွေကို apply လုပ်မယ်ဆိုရင်တော့ အဲ့ဒီအပေါ်မှာသတ်ရောက်မှုရှိမှာမဟုတ်ပါဘူး။verticle layout ပြောင်းတဲ့အခါမှာ reverse ဆိုတဲ့ property ကိုထည့်သုံးမယ်ဆိုရင်တော့ browser ထဲမှာ အောက်ကနေအပေါ်အတိုင်းပဲစီပေးမှာဖြစ်ပါတယ်။အဲ့တာတွေကိုကျွန်တော်တို့ navigation responsive အတွက်အစားထိုးအသူံးပြုနိုင်ပါတယ်။သေးငယ်တဲ့မျက်နှာပြင်ပေါ်မှာlayout ကို column တစ်ခုနဲ့ပေါ်ပြပြီးတော့ ကျယ်တဲ့မျက်နှာပြင်ပေါ်မှာတော့ row နဲ့ဖော်ပြနိုင်တာပဲဖြစ်ပါတယ်။
