    LEARNING CSS Selectors

  • Introduction

    အခုကျွန်တော်တို့ basic selector concepts တွေအကြောင်းကိုလေ့လာသွားမှာဖြစ်ပါတယ်။ကျွန်တော်တို့ CSS basics မှာ type,ID,class တွေနဲ့ decendant selectors တွေအကြောင်းကိုလေ့လာခဲ့ကြပြီးဖြစ်ပါတယ်။Selector တွေဟာ CSS တွေရဲ့အရေးအကြီးဆုံးအပိုင်းတွေနဲ့ powerful ဖြစ်တဲ့အပိုင်းတွေပဲဖြစ်ပါတယ်။ဘာ့ကြောင့်လဲဆိုတော့ သူတို့က Html element တွေကို target လုပ်နိုင်ပြီး style တွေသတ်မှတ်နိုင်လို့ပဲဖြစ်ပါတယ်။ဒီတစ်ခါမှာတော့ကျွန်တော်တို့ attribute selectors, child selectors, selector combinators တွေနဲ့ အသုံးဝင်တဲ့ pseudo-classes တွေအကုန်လုံးကိုလေ့လာရမှာဖြစ်ပြီး core concepts ကိုပိုပြီးလေ့လာသွားမှာဖြစ်ပါတယ်။ selectors တွေရဲ့ထူးခြားချက်တွေ ရိုးရှင်းတဲ့ expressions တွေနဲ့အသုံးပြုနိုင်ပုံတွေ elements တွေကိုပေါင်းပြီးတော့ target လုပ်တာတွေကိုကျွန်တော်တို့ပိုပြီးမြင်နိုင်မှာပဲဖြစ်ပါတယ်။ ကျွန်တော်တို့ special selectors တွေကိုအသုံးပြုမှာဖြစ်ပြီးတော့ pseudo-elements လို့ခေါ်ပါတယ်။CSS ထဲကနေမှ page ထဲကို content နဲ့ virtual elements တွေထည့်ဖို့အတွက်ပဲဖြစ်ပါတယ်။အဲ့တာတွေကိုလေ့လာပြီးတဲ့အခါမှာကျွန်တော်တို့ CSS selectors တွေရဲ့အလုပ်လုပ်ပုံတွေကောင်းမွန်ပြီး flexibility ဖြစ်တာတွေကိုနားလည်သွားမှာပဲဖြစ်ပါတယ်။ကျွန်တော်တို့ရဲ့အမြဲတမ်းအသုံးပြုနေကြဖြစ်တဲ့ CSS tool kit တစ်ခုကိုအသုံးပြုပြီးလုပ်ဆောင်ရမှာပဲဖြစ်ပါတယ်။selector တွေကိုကျွန်တော်တို့ကအမြဲတမ်းမှတ်မိနေစရာမလိုအပ်ပါဘူး။

  • Using Selectors

    Html elements တွေကအမျိုးမျိုးသော attributes အမျိုးအစားတွေကိုလက်ခံနိုင်ပါတယ်။ဥပမာ id attribute, class attribute, href နဲ့ title attribute တွေပဲဖြစ်ပါတယ်။ကောင်းမွန်တာတစ်ခုကတော့ CSS မှာ element မှာရှိတဲ့၎င်းရဲ့ဘယ် attributes နဲ့ values ကို target ပြုလုပ်နိုင်တာပဲဖြစ်ပါတယ်။အဲ့တာတွေကတော့ elements တွေမှာ style လုပ်ဖို့အတွက်အမှန်တကယ်အသုံးဝင်တာပဲဖြစ်ပါတယ်။ကျွန်တော်တို့ဥပမာထဲမှာပြထားသလိုလက်တွေ့ရေးတဲ့အခါမှာ elements ပေါ်မှာအခြေခံပြီးတော့အဲ့တာရဲ့ attributes နဲ့ values တွေပေါ်မှာ target ထားပြီ attributes selector တွေကိုအသုံးပြုနိုင်ပါတယ်။ကျွန်တော်တို့ဟာ attribute selector တွေကိုရေးတဲ့အခါမှာ square bracket ထဲမှာ attribute nameကိုရေးပြီးအဲ့ဒီနောက်မှာတော့ curky braces ရေးပြီး property တွေ value တွေထည့်ရေးပေးရမှာပဲဖြစ်ပါတယ်။elements တွေကိုတိတိကျကျသတ်မှတ်နိုင်ဖို့အတွက် ကျွန်တော်တို့က element name တွေနဲ့ attributes တွေကိုပေါင်းပြီးတိတိကျကျခေါ်ပြီးသတ်မှတ်ပေးလို့ရပါတယ်။elements selectors တွေနဲ့ကွာဟမှုသိပ်မရှိပေမယ့် attributes တူတာတွေကိုပေါင်းပြီး styles တွေသတ်မှတ်နိုင်ပါတယ်။ 

    
/*======================
Using Selector
======================*/                                 
form[class="form-contact"] {
    padding: 20px 24px;
    background: #f4f7f8;
}

div[id="container"] {
    max-width: 500px;
    margin: auto;
}

input[placeholder] {
    background: #fdfee6;
}

    ဒီတစ်ခါမှာတော့ကျွန်တော်တို့ အပေါ်မှာပြခဲ့တဲ့ workspace ပေါ်မှာထပ်ပြီး attributes selector ကိုသုံးပြီးပြုပြင်ပေးမှာဖြစ်ပါတယ်။buttton တွေ submit တွေပေါ်မှာ Mouse pointer ရောက်သွားတဲ့အခါပြောင်းသွားတာကိုဥပမာထဲမှာထပ်ရေးပြမှာဖြစ်ပါတယ်။အဲ့တာကြောင့်ကျွန်တော်တို့က attributes တွေကို selector တွေနဲ့တွဲပြီးတိကျအောင်ခေါ်ပေးမှာပဲဖြစ်ပါတယ်။selector တွေကိုအတူတူပေါင်းရေးတဲ့ အခါမှာတော့ comma ခံပြီးရေးပေးရပါတယ်။အဲ့တာတွေရဲ့ အတွင်းထဲမှာတော့ pointer ကိုပြောင်းမှာဖြစ်တဲ့အတွက် courser ဆိုတဲ့ attribute နောက်မှာ pointer ဆိုတဲ့ value ကိုထည့်ပေးရမှာပဲဖြစ်ပါတယ်။အောက်ပါဥပမာထဲမှာကျွန်တော်တို့cursor အတွက်ရေးပြထားတာကိုလေ့လာနိုင်ပါတယ်။ 
                        
input[type="button"],
input[type="reset"],
input[type="submit"] {
    cursor: pointer;
}

a[target='_blank'] {
    color: #39add1;
    text-decoration: none;
    border-bottom: 1px dotted;

  • DRY

    အခုကျွန်တော်တို့ဆက်ပြီးလေ့လာမှာကတော့ကြီးမားတဲ့ benefit တစ်ခုဖြစ်ပြီး style တွေကိုကြေငြာတဲ့အခါမှာထတူညီတာတွေထပ်မနေစေရန်အတွက်ရှောင်ရှားပေးတာပဲဖြစ်ပါတယ်။css ထဲမှာ(သို့)ရေဘူရအားဖြင့် web development ထဲမှာဆိုရင်အဲ့တာတွေကအသုံးပြုဖို့အတွက်အကောင်းဆုံးပဲဖြစ်ပြီး DRY လို့ခေါ်ပါတယ်။အရှည်ကတော့ Don't Repeat Yourself ဖြစ်ပြီးအဓိကရည်ရွယ်ချက်ကတော့တူညီတဲ့ code တွေကိုထပ်ခါရေးထားတွေကိုရှောင်ရှားဖို့ပဲဖြစ်ပါတယ်။ကျွန်တော်တို့ CSS ထဲမှာ stylesheet တစ်ခုကိုအကြိမ်ရေအများကြီးရေးထားတာရှိတယ်ဆိုရင်ကျွန်တော်တို့ပြန်ပြီးပြုပြင်တဲ့အခါမှာထိန်းသိမ်းဖို့ခက်ခဲပါလိမ့်မယ်။အဲ့တာတွေပြောင်းဖို့ဆိုရင်ကျွန်တော်တို့ rule တစ်ခုအစားကျွန်တော်တို့ rules တွေအများကြီးလိုအပ်ပါလိမ့်မယ်။အဲ့တာတွေတင်မကပဲ stylesheet ရဲ့size ကလည်းကြီးမားနေပါလိမ့်မယ်။ကျွန်တော်တို့ရဲ့ Page ကို browser ပေါ်မှာrun တဲ့အခါမှာအချိန်တွေပိုပြီးကြာနေပါလိမ့်မယ်။အဲ့တာကြောင့်ကျွန်တော်တို့ကပိုပြီးကောင်းတဲ့ idea တစ်ခုဖြစ်တဲ့ property တစ်ခုစီကို value တွေအများကြီးနဲ့တစ်ခါပဲသတ်မှတ်ပေးမှာဖြစ်ပါတယ်။IDတွေကပြန်ပြီးအသုံးပြုလို့မရဘူးဆိုတာကိုကျွန်တော်တို့သိကတည်းက DRY methodology တွေက css တွေနဲ့အကောင်းဆုံးအလုပ်လုပ်ပါတယ်။ကျွန်တော်တို့ Page ထဲမှာတူညီတဲ့ STYLE တွေကိုသတ်မှတ်ချင်တဲ့အခါ DIY method ကိုအသုံးပြုပါတယ်။DRY method အလုပ်လုပ်ပုံကတော့ကျွန်တော်တို့ style ပြုလုပ်မဲ့ elementsတွေကို class တူညီတာကိုပေးခဲ့ပြီး CSS ထဲမှာအဲ့ဒီ class ကိုခေါ်သုံးတာပဲဖြစ်ပါတယ်။elements တွေမှာ class တွေသတ်မှတ်တာရှိပြီးသားဆိုလျှင်လည်းထပ်ပြီးသတ်မှတ်နိုင်ပါတယ်။class တွေအများကြီးသတ်မှတ်တဲ့နည်းကတော့ ပထမသတ်မှတ်ထားတဲ့ class nameရဲ့နောက်မှာ space ထားပြီးကျွန်တော်တို့ထားချင်တဲ့ class name ကိုထားပြီးသတ်မှတ်နိုင်ပါတယ်။အဲ့ဒီလိုတွေအသုံးပြုချင်းက css file size ကိုလည်းလျော့ချပေးပြီးပိုပြီးမြန်ဆန်စေပါတယ်။ဥပမာထဲမှာဆိုရင် border radius styles တွေ button တွေကိုစုပြီးသတ်မှတ်ပြထားပါတယ်။

  • Combinators

    Combinators တွေက selectors တွေထဲမှာရှိတဲ့ elements တွေရဲ့ဆက်နွယ်မှုတွေကိုရှင်းပြတာပဲဖြစ်ပါတယ်။ကျွန်တော်တို့ဟာအချိန်တိုင်းမှာ selector တစ်ခုနဲ့ တစ်ခုသို့မဟုတ်တစ်ခုထက်ပိုတဲ့ elements တွေကိုခေါ်ပြီးအသုံးပြုကြပါတယ်။ combinator ကိုကျွန်တော်တို့အသုံးပြုဖို့လိုအပ်ပါတယ်။ပြီးတော့ကျွန်တော်တို့ decendant selectors တွေရေးတဲ့အခါမှာ CSS basics တွေထဲမှာ combinators တွေကိုအမှန်တကယ်အသုံးပြုပါတယ်။ဘာ့ကြောင့်လဲဆိုတော့ selectors နှစ်ခုကြားထဲမှာရှိ white space ကလည်းပဲ combanitors ကိုဆုံးဖြတ်ပေးပါတယ်။decendant elements တွေကိုပြစ်မှတ်ထားပြီးအသုံးပြုတဲ့အခါ combinators တွေက element ရဲ့ချိတ်ဆက်နေ target ပြုလုပ်ထားတာတွေကို flexibility ဖြစ်စေပါတယ်။ကျွန်တော်တို့တွေက greater than sign, plus sign, tilde symbol တွေကိုအသုံးပြုပြီး selector ကိုပိုပြီးတိကြအောင်ပြုလုပ်နိုင်ပါတယ်။ပထမအဆင့်အနေနဲ့ CSS selectorမှာ greater than sign က child combinator တစ်ခုကိုရည်ညွှန်းတာပဲဖြစ်ပါတယ်။ဘာ့ကြောင့်လဲဆိုတော့အဲ့တာကအခြား elements တစ်ခုရဲ့ direct children ဖြစ်တဲ့ element ကိုရည်ညွန်းတာပဲဖြစ်ပါတယ်။direct child ဆိုတာကတော့ parent element ထဲမှာတိုက်ရိုက် nested ဖြစ်တဲ့ elements တွေပဲဖြစ်ပါတယ်။နောက်ထပ် combinator တစ်ခုဖြစ်တဲ့ plus sign ကိုအသုံးပြုတဲ့ selector ကိုလေ့လာရမှာဖြစ်ပါတယ်။plus sign combinator ကို adjancent sibling combinator လို့ခေါ်ပါတယ်။ပြီးတော့ selectors တွေကိုအသုံးပြုခြင်းကို adjacent sibling selectors လို့ခေါ်ပါတယ်။ဘာ့ကြောင့်လဲဆိုတော့ သူတို့က page ပေါ်မှာ element တစ်ခုကိုtarget ထားပြီးနောက်ထပ်နီးစပ်တဲ့ elements တစ်ခုနဲ့ပဲဖြစ်ပါတယ်။ကျွန်တော်တို့ ဥပမာထဲမှာ combinator တွေကိုအသုံးပြုပြထားပါတယ်။နောက်ဆုံး combinator တစ်ခုကတော့ ကျွန်တော်တို့ tilde symbol ကိုအသုံးပြုတာပဲဖြစ်ပါတယ်။အဲ့တာကို general sibling combinator လို့ခေါ်ပါတယ်။ဘာ့ကြောင့်လဲဆိုတော့ အဲ့တာက siblings ဖြစ်တာတွေကို target ထားတာပဲဖြစ်ပါတယ်။ဥပမာထဲမှာအဲ့တာတွေရဲ့အလုပ်လုပ်ပုံတွေကိုဖော်ပြပေးထားပါတယ်။အချုပ်အားဖြင့်ပြောရမယ်ဆိုရင် ကျွန်တော်တို့ specific child နဲ့ sibling selectors တွေကို combinators     ပြီးခဲ့တဲ့သင်ခန်းစာတုန်းက structural pseudo-classes တွေနဲ့ကျွန်တော်တို့ပထမနဲ့ နောက်ဆုံး child elements ကိုဘယ်လို target လုပ်မယ်ဆိုတာကိုလေ့လာခဲ့ပြီးဖြစ်ပါတယ်။နောက်ထပ်ကျွန်တော်တို့ nth-child pseudo -class ကဘယ်လိုအလုပ်လုပ်တယ်ဆိုတာရယ် child elements တွေရဲ့မည်သည့် combination ကိုမဆို target လုပ်နိုင်တယ်ဆိုတာတွေကိုမြင်တွေ့ရမှာဖြစ်ပါတယ်။nth-child ဟာအသုံးဝင်ပြီးတော့ pseudo-class တည်ဆောက်ပုံတွေအတွက် Powerful ဖြစ်ပါတယ်။ဘာ့ကြောင့်လဲဆိုတော့ parent element အတွင်းထဲမှာ မည့်သည့်နေရာကိုမဆိုအခြေခံပြီးတော့ elements တွေကို target လုပ်နိုင်ပါတယ်။ဥပမာထဲမှာတော့ကျွန်တော်တို့ list တွေနဲ့ဥပမာပြထားပါတယ်။ကျွန်တော်တို့ nth-child selector ကိုတည်ဆောက်တဲ့အခါ li ဆိုတဲ့ list item ကိုရေးပြီးအဲ့ဒီနောက်မှာ nth-child ဆိုတာလိုက်ရပါမယ်။အဲ့တာရဲ့နောက်မှာမှ parentheses လိုက်ပေးရပါမယ်။nth-child က functional like syntax ကိုအသုံးပြုပါတယ်။အဲ့တာက parentheses ကြားထဲမှာရှိတဲ့ value တွေ pass လုပ်ဖို့ခွင့်ပြုပါတယ်။အဲ့ဒီ values က argument တစ်ခုကိုရည်ညွှန်းပြီးတော့ selector ကိုဘယ်ဟာကို pass လုပ်မယ်ဆိုတာကိုဖော်ပြတာပဲဖြစ်ပါတယ်။ပထမအနေနဲ့ကျွန်တော်တို့ argument ဖြစ်တဲ့ odd သို့ even ကိုထည့်နိုင်ပါတယ်။အဲ့ဒီနေရာမှာ ကျွန်တော်တို့ထည့်ချင်တဲ့ item number ကိုထည့်ပြီးလည်း apply လုပ်နိုင်ပါတယ်။ odd ဆိုတာကိုထည့်ပြီးအသုံးပြုမယ်ဆိုရင် list items တွေရဲ့ odd number တွေအတိုင်းအလုပ်လုပ်ပါတယ်။even ကိုထည့်ပြီးအသုံးပြုမယ်ဆိုရင်တော့ list items တွေရဲ့ even number တွေအတိုင်းအလုပ်လုပ်ပါတယ်။နောက်ထပ်ကျွန်တော်တို့ကနောက်ထပ် integer Number တွေကိုလည်းထည့်နိုင်ပါတယ်။nht child ဟာအမှန်တကယ် powerful ဖြစ်ပြီးတော့နောက်ထပ်ရှုပ်ထွေးတဲ့ combination တွေလည်းထပ်ထည့်နိုင်ပါတယ်။ဥပမာ (an+b) ဆိုတာကိုထည့်ရေးတဲ့အခါ a နဲ့ b ကnumber တွေကိုကိုယ်စားပြုပြီးတော့ n ဆိုတာကတော့မပြောင်းလဲတဲ့ value တစ်ခုပဲဖြစ်ပါတယ်။plug sign ကတော့တန်ဖိုးတွေကိုပေါင်းပြီးတော့ list item ထဲမှာသတ်မှတ်ပေးတာပဲဖြစ်ပါတယ်။ minus n ဆိုပြီးလည်းထည့်သုံးနိုင်ပြီးအဲ့ဒီအခါမှာတော့ list item ထဲမှာလျှော့ပြီးသတ်မှတ်သွားမှာဖြစ်ပါတယ်။ကျွန်တော်တို့ဥပမာထဲမှာအဲ့တာတွေအကုန်လုံးကိုလေ့လာနိုင်ပါတယ်။
